## Key Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| ESOP = 1B EXP = 10% equity | Turns gamification into real financial stake. Quit = lose it is the retention mechanic. | Separate share register (complex), vesting cliff system (already existed, too slow) |
| Historical EXP counts 1:1 | Honours past effort. Natural dilution (growing denominator) handles gap for newcomers. | Fresh start for all (fair but demoralises existing staff), capped historical (arbitrary) |
| Rehire = fresh start, 50% retain Creator-only | Zero tolerance for churn + Creator discretion for special circumstances | Auto-restore (no consequence for quitting) |
| Company valuation: 8B VND | 4x revenue multiple. Billiards 2x (mature) + Kitchen 4x (growth) + 1B synergy. Defensible today, room to grow. | 6B (too conservative), 12B (premature without Y2 kitchen data) |
| Mission 015 after 006 not parallel | 006 and 015 touch same layer. Fragmented classifier + tool-use = debug two things at once. | Parallel (faster but creates technical debt) |
| Project Nova: hosted SaaS | We maintain one codebase, push fixes instantly, no client server management. Soul + data with client via API keys. | Self-hosted (support nightmare, version fragmentation) |
| Tier 1 connectors first | API integrations cover 80% of use cases, zero installation friction for clients. | Browser extension (higher friction), local companion (requires install) |
